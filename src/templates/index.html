Introduction Assignment4

This webpage can query an existing database located on a local server. For the creation of the triple store stardog is used. This particular webpage can query from a triple store about different recipes based on difficulty of the recipe, origin and the ingredients needed. The query can be performed with and without inferencing. When enabling inferencing the reasoner of stardog is enabled. Stardog is an OWL reasoner.

Results explaination

The results for the queries are different when the reasoner is actived. The 5 examples below are shown in the video to demonstrate what the reasoner does in this triple store

  1. Recipe(Class)
  2. hasOrigin(objectProperty)
  3. hasPreperationTime(dataProperty)
  4. JapaneseRecipe(Class)
  5. BeginnerRecipe(Class)

    6. Recipe(Class)

    Without inferencing the query does not show the instances of the recipes. The reason for this is that the recipes are not instances of the Recipe class but an instance of a subclass named after the recipe name. When enabling inferencing the three instances of the recipes are displayed.

    hasOrigin(objectProperty)

    Without inferencing the query does not show the instances of the recipes nor the countries of origin. The reason for this is that without the reasoner it is not inferred that the recipe instance that belongs to a class has the same origin as that class. When enabling inferencing the three instances of the recipes are shown with the correct origin instance as the object property assertions are inferred.

    hasPreperationTime(dataProperty)

    The results for this query are the same with and without the reasoning. The dataproperty is given to the instance and thus not has to be inferred for the results.

    JapaneseRecipe(Class)

    The results for this query differ because the class that the recipe instance belongs to is only determined to be a subclass of the JapaneseRecipe class in the inferred hierarchy(in Protege).

    BeginnerRecipe(Class)

    The results for this query differ because the recipe instance is only determined to be an instance of the BeginnerRecipe class in the inferred hierarchy(in Protege).
